1. Background / Theory

Effective personal financial management is a growing challenge in the digital age. Many individuals lack accessible, user-friendly tools to track and analyse their daily expenditures, often leading to overspending and financial instability. A web-based expense tracker directly addresses this gap by offering a centralised platform where users can log transactions, categorise them, and obtain meaningful summaries of their spending behaviour.

1.2 MVC Design Pattern

The application follows the Model-View-Controller (MVC) architectural pattern, which separates concerns into three distinct layers:

  • Model – C# classes (Expense, ExpenseStore) that define data structure and business rules.
  • View – Razor .cshtml files that render HTML pages dynamically from model data.
  • Controller – C# classes (HomeController, ExpenseController) that handle HTTP requests, process input, and return appropriate views.
Motivation: By building this application students develop problem-solving skills, creativity, and a deeper understanding of web technologies—preparing them for real-world challenges in web development.

3. Code Implementation

3.1 Expense Data Model

The Expense class defines the structure of each expense record and uses Data Annotations to enforce server-side validation automatically.

Models/Expense.cs
using System.ComponentModel.DataAnnotations;

namespace ExpenseTracker.Models
{
    public class Expense
    {
        public int Id { get; set; }

        [Required(ErrorMessage = "Description is required")]
        [StringLength(100)]
        public string Description { get; set; } = string.Empty;

        [Required(ErrorMessage = "Amount is required")]
        [Range(0.01, 999999.99, ErrorMessage = "Amount must be greater than 0")]
        [DataType(DataType.Currency)]
        public decimal Amount { get; set; }

        [Required(ErrorMessage = "Category is required")]
        public string Category { get; set; } = string.Empty;

        [Required(ErrorMessage = "Date is required")]
        [DataType(DataType.Date)]
        public DateTime Date { get; set; } = DateTime.Today;

        public string? Notes { get; set; }
    }
}

3.2 In-Memory Data Store

Instead of a database, a static list simulates persistent storage for the duration of the application session—ideal for a lab demonstration.

Models/ExpenseStore.cs
public static class ExpenseStore
{
    private static int _nextId = 1;

    public static List<Expense> Expenses { get; } = new()
    {
        new Expense { Id = _nextId++, Description = "Grocery shopping",
                      Amount = 85.50m, Category = "Food", Date = DateTime.Today.AddDays(-1) },
        new Expense { Id = _nextId++, Description = "Bus pass",
                      Amount = 30.00m, Category = "Transport", Date = DateTime.Today.AddDays(-2) },
        // ... more seed data
    };

    public static readonly List<string> Categories = new()
    {
        "Food", "Transport", "Shopping", "Entertainment",
        "Bills", "Health", "Education", "Other"
    };

    public static int GetNextId() => _nextId++;
}

3.3 Expense Controller (CRUD)

The ExpenseController handles all create, read, update, and delete operations. Filtering logic is applied in the Index action before passing data to the view.

Controllers/ExpenseController.cs
public class ExpenseController : Controller
{
    // GET: /Expense — with optional filters
    public IActionResult Index(string? category, string? search,
                                  string? dateFrom, string? dateTo)
    {
        var expenses = ExpenseStore.Expenses.AsQueryable();

        if (!string.IsNullOrEmpty(category) && category != "All")
            expenses = expenses.Where(e => e.Category == category);

        if (!string.IsNullOrEmpty(search))
            expenses = expenses.Where(e => e.Description
                .Contains(search, StringComparison.OrdinalIgnoreCase));

        if (DateTime.TryParse(dateFrom, out var from))
            expenses = expenses.Where(e => e.Date >= from);

        ViewBag.Total = expenses.Sum(e => e.Amount);
        return View(expenses.OrderByDescending(e => e.Date).ToList());
    }

    // POST: /Expense/Create
    [HttpPost][ValidateAntiForgeryToken]
    public IActionResult Create(Expense expense)
    {
        if (ModelState.IsValid)
        {
            expense.Id = ExpenseStore.GetNextId();
            ExpenseStore.Expenses.Add(expense);
            TempData["Success"] = "Expense added successfully!";
            return RedirectToAction(nameof(Index));
        }
        ViewBag.Categories = ExpenseStore.Categories;
        return View(expense);
    }

    // POST: /Expense/Delete/{id}
    [HttpPost][ValidateAntiForgeryToken]
    public IActionResult Delete(int id)
    {
        var expense = ExpenseStore.Expenses.FirstOrDefault(e => e.Id == id);
        if (expense != null) ExpenseStore.Expenses.Remove(expense);
        TempData["Success"] = "Expense deleted!";
        return RedirectToAction(nameof(Index));
    }
}

3.4 Razor View – Add Expense Form

Razor views combine HTML markup with C# expressions. The form below uses Tag Helpers (asp-for, asp-action) to bind model properties and generate anti-forgery tokens automatically.

Views/Expense/Create.cshtml (excerpt)
@model ExpenseTracker.Models.Expense

<form asp-action="Create" method="post" class="expense-form">
    @Html.AntiForgeryToken()

    <div class="form-row">
        <div class="form-group">
            <label asp-for="Date" class="form-label"></label>
            <input asp-for="Date" type="date" class="form-input" />
            <span asp-validation-for="Date" class="validation-error"></span>
        </div>
        <div class="form-group">
            <label asp-for="Category" class="form-label"></label>
            <select asp-for="Category" class="form-input">
                @foreach (var cat in ViewBag.Categories)
                {
                    <option value="@cat">@cat</option>
                }
            </select>
        </div>
    </div>

    <div class="form-group">
        <label asp-for="Amount" class="form-label"></label>
        <input asp-for="Amount" type="number" step="0.01"
               class="form-input" />
    </div>

    <button type="submit" class="btn btn-primary">Save Expense</button>
</form>

5. Reflection Report

5.1 Challenges Faced

1

Razor Expression Syntax

Calling .ToString("N2") outside a cast expression rendered as literal text. Fixed by wrapping the entire expression inside @("$" + value.ToString("N2")).

2

ViewImports File Location

Placing _ViewImports.cshtml inside Views/Shared/ instead of Views/ caused tag helpers and namespaces to fail for all non-shared views, breaking CSS links and form binding.

3

HTTPS Redirect on HTTP-Only Server

UseHttpsRedirection() redirected traffic to a HTTPS port that was never started, producing an HTTP 403 error. Removed the middleware to serve the app over plain HTTP in development.

4

Dynamic vs Typed ViewBag

Using anonymous types with dynamic in ViewBag caused runtime failures inside Razor views. Introduced a concrete CategoryTotal DTO class to allow strongly-typed access.

5

Razor Option Selected Attribute

The Razor error RZ1031: tag helper must not have C# in attribute occurred when using an inline ternary for selected. Replaced with an explicit @if / else block.

6

Target Framework Mismatch

The project initially targeted net8.0 while only .NET 10 SDK was installed, causing a slow workload resolution warning. Updated the .csproj to net10.0.

5.2 Potential Improvements

  • Database Integration – Replace the in-memory store with Entity Framework Core + SQLite/SQL Server for true data persistence across sessions.
  • User Authentication – Add ASP.NET Core Identity so each user has a private set of expense records.
  • Budget Alerts – Allow users to set monthly spending limits per category and receive visual warnings when approaching or exceeding the limit.
  • Interactive Charts – Integrate Chart.js to render animated pie/doughnut and line-trend charts instead of CSS-only bar charts.
  • Data Export – Add CSV / PDF export functionality so users can analyse their data in Excel or share it with an accountant.
  • Advanced Analytics – Display week-over-week and month-over-month spending trends, and predict future expenditure using simple moving averages.

5.3 Discussion on Results

The web-based expense tracker successfully met all primary objectives defined in the problem statement. The application provides users with a clean, responsive interface to log daily expenses, assign them to meaningful categories, and view both detailed records and high-level summaries on the dashboard.

The category bar chart offers an immediate visual understanding of where money is being spent, while the filter and search capabilities on the expenses table allow users to quickly isolate specific transactions by date range or keyword. The edit and delete functionality ensures records remain accurate over time.

From a technical standpoint, the project reinforced key ASP.NET Core MVC concepts including routing conventions, model binding, data annotations for validation, anti-forgery tokens for security, and the Razor view engine. The CSS design system built with custom properties and CSS Grid demonstrates how a consistent, scalable stylesheet can be authored without any third-party framework.

Overall, the application fulfils the brief and provides a solid foundation upon which real-world features—persistent storage, user accounts, and rich analytics— could be built in a production context.
